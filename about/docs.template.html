Uvod

PhoneAPP je aplikacija koja služi za pregledavanje mobilnih uređaja, pretraživanje, sortiranje istih. Aplikacija se bazira na posebno izrađenom API servisu koji je namjenjen kako bi što bolje komunicirao s Angular frameworkom u pozadini, ali nešto više o tome kasnije.

Tehnologije

  • Angular
  • jQuery UI
  • jQuery
  • Ajax
  • PHP
  • PDO Klasa
  • Angular template engine
  • Bower
  • Node
  • Model based podatci

Backend dio

Includes

  • AccessControl - klasa za provjeru pristupa nekoj stranici
  • Criteria - klasa koja služi za stvaranje upita nad bazom, serijalizira parametre i preventira SQL injection, JSON se koristi za prijenos preko GET parametra. Klasa ima implementirane metode za prihvat condition, select, limit, group, join upita i to sve preko parametriziranih upita.
  • Helpers - Klasa sadrži pomocne metode, kao što su time() za virtualno vrijeme te ostale koje se često koriste
  • Router - Klasa koja služi kao sučelje parametrima iz URL-a i http headera. Mogli bi reći kao malo sigurniji $_GET. Sadrži statičke metode za pristup parametrima i izvođenju operacija nad URI-em
  • Service - Bazna klasa za sve servise koji se pozivaju iz frontend dijela, a vuku podatke iz baze, klasa koristi Criteria i Router klase za pretvaranje parametara u upite koji se onda koriste za dohvačanje bilo kojeg podatka iz baze.

Models

  • Database - klasa proširuje PDO klasu PHP-a i služi za izvršavanje svih upita nad bazom. Parametre dobiva preko servisa i upita pripremljenih sa Criteria klasom
  • ActiveRecord - Svi ostali modeli koji imena imaju po tablicama u bazi extendaju ovu klasu. Ona sadrži bazne metode za CRUD operacije, tj. spremanje, brisanje itd. Isto tako sadrži staticki element formData koji se preko servisa koristi za stvaranje i dinamičko generiranje CRUD formi u frontendu.
  • Model - pomocna klasa. Svaka klasa tipa ActivaRecord ima staticku instancu ove klase. Služi za dinamičko dohvacanje Podataka, a bazira se na MVC arhitekturi. Osnovne metode su findeOne, tj. po PK i druga findAll koja prima argument tipa Criteria
  • Device - izdvajam ovu klasu jer ona koristi i datoteke koje se vežu za nju, tj. slike mobilnog uređaja.
  • User - klasa koja uz klasu UserRole zajedno s AccessControl klasom služi za menađiranje pristupa raznim djelovima aplikacije.

Protected

Folder je zašticen .htaccess datotekom te se unutar njega nalazi servis koji ispisuje sve korisnike.

Services - Data

Unutar ovog foldera se nalati sva magija oko servisa. Naime u frontend dijelu kreirao sam Angularove servise koji se onda pozivaju na ove u backend-u. Svaka datoteka inicijalzira objekt tipa Service i nakon toga pomocu ostalih klasa koje sam opisao gore procesira upite i vraca nazad json formatirani string koji se u frontendu koristi za prikaz i generiranje podataka.

Bazne datoteke backend

  • header.php - ova datoteka se brine da sve potrebne datoteke i klase budu učitane kod svakog poziva API-a
  • config.php - razne postavke aplikacije, te rute i pravila pristupa koje odgovaraju onima u configu frontend dijela.

Frontend dio

Modularni pristup

Htio sam kroz ovu aplikaciju složiti modularnu arhitekturu koja bi bila lako prenosljiva. Zato su više manje svi view-i u različitim modulima i komponentama koje žive same za sebe. Ovakav pristup dakako uzima više vremena i zato me to u jednu ruku koštalo završavanja funkcionalnosti kao što su statitstike i slično.

Modul servisa (Core)

Unutar core/services mape se nalazi services modul kojeg ekstendaju sve service komponente unutar istoga. Svaka komponenta ima dvije metode. Get i Query. Razlika je u tome da Query metoda ocekuje kao odgovor array, dok Get želi object što je u početku bilo malo zbunjujuce za pohvatati. Kroz GET parametre servisi šalju parametre zahtjeva za pozadinske upite. Ti upiti se tada pretvaraju kroz Service objekt u formatirani upit kakav prima moj API, a to je Criteria objekt. Nakon toga se poziva metoda findAil s tim argumentom.

Ostali view moduli

  • login - služi za prijavu i prikaz forme. Forma se naravno dinamički submita.
  • registration - ista stvar koristi se za registraciju novog korisnika
  • device-list - Prikaz, ispis, pretraživanje uređaja.
  • device-list - Detaljni prikaz mobitela uz galeriju slika.

Bazne datoteke frontend

  • index.html - koristi se kao početna datoteka, u njoj su inlcude-ane sve komponente i pripadajuci moduli koji su potrebni u aplikaciji.
  • app.animations - ova datoteka je trebala biti malo veća, ali za sada se koristi samo za animiranje kod prelgeda galerije slika mobitela
  • app.config - ovdje se definiraju globalni $rootScope parametri. Ovdje isto tako kod svakog pokretanaj aplikaciji i kod svake promjene rute se provjerava u API-u da li je korisnik prijvaljen i da li ima pristup nekoj stranici.
  • app.module - bazni modul aplikacije
